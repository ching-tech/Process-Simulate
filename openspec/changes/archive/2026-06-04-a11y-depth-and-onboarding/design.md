## Context

接續 `ux-a11y-polish-pass` 的基線，本期處理「基線之後」會被真實使用者踩到的五個點。全部加成式、不改引擎、不改既有外觀；色盲與單位提示連顏色都不動，只加冗餘資訊。

## Goals / Non-Goals

**Goals**
- 讓色盲使用者也能讀懂 AGV 載貨狀態。
- 讓鍵盤/報讀器使用者在彈窗與動態數值上不迷路。
- 讓新使用者發現既有的隱藏互動。
- 讓秒/分單位落差不再造成輸入誤解。

**Non-Goals**
- 不改顏色語意（色盲修正用「加形狀」而非「換色」）。
- 不做完整 WAI-ARIA 全站稽核（聚焦這五點）。
- 不改參數的實際單位（仍以秒儲存；只加顯示提示）。

## Key Decisions

### D1. 色盲：加形狀，不換色
AGV 維持綠/橘（與站別框語意一致），但在車身或貨物點疊一個**形狀/符號**區分：載滿框 = 實心方塊 `■`、載空框 = 空心方框 `□`、空手 = 無標記。如此色覺正常者看顏色、色盲者看形狀，兩者皆可判讀。圖例（`abf-hdr2`）同步補上形狀說明。

```
載滿框  ●綠 + ■   載空框  ●橘 + □   空手  淡灰(無)
        ↑顏色 ↑形狀（色盲憑此分辨）
```

### D2. Focus trap：輕量實作，與既有 Esc 整合
開啟彈窗時：記住 `document.activeElement`（觸發者）→ 把焦點移到彈窗第一個可聚焦元素 → 監聽彈窗內 `keydown` Tab，在首/末元素間循環。關閉時：還原焦點到觸發者。沿用上期已加的全域 `Esc` 關閉邏輯。實作為通用小工具 `trapFocus(container)` / `releaseFocus()`，套用到 agv-board、line-wizard、完成 overlay、confirm-ov。

### D3. aria-live：只播里程碑，避免吵
新增視覺隱藏的 `#sr-live`（`aria-live="polite"` `aria-atomic="true"`）。`announce(msg)` 寫入其 textContent。**只在事件性里程碑**呼叫——達成目標、出現「新」瓶頸（去重，不每幀念）、套用參數重跑、切換主題——不綁定每秒更新的時鐘/進度，以免報讀器被洗版。

### D4. Coachmark：首次一次性，之後常駐入口
首次進入模擬（localStorage 無 `sim-onboarded`）時，顯示三則指向式提示（點站別／滾輪縮放／拖曳平移），可「知道了」關閉並寫旗標。之後在 header 放常駐 `?` 鈕可隨時再叫出。提示為純疊層，不阻擋既有操作（可直接點掉）。

### D5. 單位提示：顯示換算，不改儲存
`fieldHTML()` 對「秒」類欄位（`unit` 為秒、或 key 屬製程/行走時間）在標籤旁加灰字 `≈ N 分`，隨輸入即時更新。底層仍以秒儲存與計算；mm:ss 欄位維持原樣。

## Risks / Trade-offs

- **形狀疊加的視覺擁擠**：AGV 圖示小，符號需夠小且高對比；tasks 含縮放下可讀性檢查。
- **Focus trap 邊界**：動態內容（精靈可新增站別列）需在內容變動後重算可聚焦集合；以「每次 Tab 時即時查詢」而非快取避免失準。
- **aria-live 去重**：瓶頸播報需記錄「已播報的瓶頸站集合」，僅在新增時念。
- **Coachmark 干擾**：必須可一鍵關閉且不擋點擊；預設只出現一次。

## Migration / Rollout

無資料遷移。五項可獨立上線、互不相依。Coachmark 旗標與既有 localStorage 鍵（`sim-theme` 等）獨立。色盲與單位提示為純加成顯示，可隨時移除而不影響其餘。
