## Context

模擬輸出散落在多個畫面（header 進度、即時統計、AGV 看板、站別 tooltip），缺一個「帶得走、比得了、看得懂下一步」的出口。本 change 在不動引擎、不新增相依的前提下，補上 export / compare / advise 三件事。資料皆已存在，重點是彙整與呈現。

## Goals / Non-Goals

**Goals**
- 讓使用者把一次模擬的完整結果帶走（資料用 CSV、簡報用列印摘要）。
- 讓多個方案能並排比較，標出最佳。
- 把「哪裡塞」接到「該怎麼調」。

**Non-Goals**
- 不改模擬引擎、物理或參數語意。
- 不引入截圖/圖表/PDF 函式庫（PNG 用原生 Canvas 自繪，不依賴 html2canvas）。
- 不做雲端儲存或多人協作。

## Key Decisions

### D1. 匯出格式：一鍵 PNG 圖片報告（Canvas 自繪，零相依）
使用者要的是「一鍵存成可直接貼進報告/訊息的圖片」。做法：在離屏 `<canvas>` 上自繪一張排版好的報告，再 `canvas.toBlob()` 下載 PNG。不引入 `html2canvas`（避免相依與字型/汙染風險），改用原生 2D Context 自繪，內容與資料完全可控。

```
結果 ──▶ exportResultPNG()
         1. 建離屏 canvas（依 devicePixelRatio 放大 → 高解析不糊）
         2. 繪：標題列(LOGO+線別+日期時間) → 摘要指標 → 最重瓶頸
              → AGV 稼動概要(群組稼動條) → 內嵌 UPH 趨勢圖(drawImage #uc)
              → 頁尾(擎添工業 + 產生時間)
         3. canvas.toBlob(png) → 下載 report-<線>-<YYYYMMDD-HHMMSS>.png
```

報告版面（單張直式，約 1000×720 @2x）：
```
┌────────────────────────────────────────────┐
│ [LOGO]  欣興電子 ABF 線 模擬報告   2026-06-04 │
├────────────────────────────────────────────┤
│  完成 / 目標   平均 UPH   模擬時間   達標 ETA  │
│   3,820/4,000    128       7h32m     +0h28m  │
├────────────────────────────────────────────┤
│  最重瓶頸：壓機（稼動 98%）                    │
│  AGV 稼動： G1 ▓▓▓▓░ 88%  G2 ▓▓▓░ 76% ...    │
├────────────────────────────────────────────┤
│  [ 內嵌 UPH 趨勢圖（取自 #uc canvas） ]        │
├────────────────────────────────────────────┤
│  擎添工業 Ching Tech · 產生於 2026-06-04 14:22 │
└────────────────────────────────────────────┘
```

**關鍵技術點**
- **不糊**：`canvas.width = W*dpr; canvas.height = H*dpr; ctx.scale(dpr,dpr)`，繪圖座標仍用 CSS 像素。
- **不被汙染（taint）**：LOGO 為 inline base64（同源）、`#uc` 為同源 canvas，`drawImage` 後 `toBlob` 不會拋安全錯誤。
- **跨平台**：Canvas + toBlob 在瀏覽器與 Tauri WebView2 皆原生支援。
- **CSV 為可選次要**：資料再分析需求以一個輕量 `exportResultCSV()`（UTF-8 BOM）作為次要選項；PNG 為主要、預設的「匯出結果」。

### D2. 比較表用「釘選快照」累積，與即時比較卡分工
即時比較卡（`showCompareCard`）保留——它解決「我剛改的參數讓即時數字怎麼變」。新的比較**表**解決「A/B/C 三方案哪個好」：使用者按「📌 釘選目前結果」把一個 `snapshotStats()` + 情境名 + 關鍵參數存入 `pinnedScenarios[]`（記憶體陣列，session 內），表格並排各欄，每列數值標出最佳（UPH 高佳、ETA 低佳、稼動適中）。釘選資料純呈現，不影響模擬。

```
              📌方案A   📌方案B   📌方案C
UPH            120      135✓     128
ETA            8.2h     7.1h✓    7.8h
最重瓶頸        壓機      —✓       換框
AGV稼動         88%      76%      82%✓
```

### D3. 瓶頸建議用既有累計 + 設備數對應表，給「方向」不誇大
讀既有瓶頸滯留統計，取滯留最久的站，對應其設備數參數（如壓機→`abfPressN`）。建議文案保守：「壓機為主要瓶頸（稼動 X%）。可試：該站 +1 台後重跑比較。」提供按鈕「套用 +1 並重跑」——即改對應參數值、`applyP()`、自動釘選前後兩欄。不保證數字、不自動連加，避免過度承諾。

## Risks / Trade-offs

- **PNG 中文字型**：Canvas 文字依系統字型；以 `'Segoe UI', system-ui, sans-serif`（與全站一致）繪製，確保中文正常；tasks 含兩環境驗證渲染。
- **Canvas 汙染風險**：LOGO 與 `#uc` 皆同源，`toBlob` 不應拋錯；tasks 含明確驗證下載成功。
- **CSV（次要）中文編碼**：加 UTF-8 BOM 確保 Excel 正確顯示中文。
- **建議的設備對應**：自訂線的站別 → 設備數參數對應需從 blueprint 推導；無法對應時退為「指出瓶頸站」但不給 +1 按鈕。
- **比較表僅 session 內**：重整後消失；本期不持久化（可後續再加），以免擴大範圍。

## Migration / Rollout

無資料遷移。三項可獨立上線；PNG/CSV 匯出與比較表互不相依。瓶頸建議若無法對應設備參數則自動降級為純文字提示。
